Шаблон: Фабрика (Factory Method)
=============================================================
Цель:

Создание интерфейса который создает обьект. При этом, выбор того, экземпляр
какого класса создавать остеться за классом, которые имплементирует данный
интерфейс.
===============================================================
Для чего используеться:

Для делегирования создания экземпляров, другому классу.

===============================================================
Пример использования:

-зарние извесно, экземпляры, какого класса нужно будет создавать;
- класс спроэктирован таким образом, что создаваемые им обьекты имеют свойства
определенного класса.

===============================================================

Применимость
Фабричный метод определяет код производства продуктов от остального кода,
который эти продукты используют
Благодоря этому код производства можна розширять не трогая основной. Для этого
нужно создать новый подкласс и определить в нем фабричный метод, возвращая оттуда
экземпляр нового продукта.

Когда вы хотите дать возможность пользователям расширить части вашего офреймворка

Пользователи могут расширять классы вашего фреймворка через наследование.
Для этого создающие классы должны иметь конкретные создающие методы, которые
можно определить

===================================================================

Шаги реализации

1. Ппривесте все создаваемые продукты к общему инерфейсу.
2. В классе, кторый производит продукты, создать пустой фабричный метод
в качестве возвращаемого типа указать общий интерфейс продукта
3. Пройтись по коду класса и найти все участки, создающие продукты. почередно
заменить эти участки вызовами фабричного метода, пренося в него код создания различных
продуктов.
4. Для каждого типа продукта завести подклас и переопределить в нем фаьричный
метод. Переместить туда код создания соответствующего продукта из суперкласса.
5.Если создавемых продуктов сильно много для существующих подклассов создателя,
можна продумать о введении параметров в ФАБРИчный метод, которые позволяют
возвращать различные продукты в пределах одного подкласса.
6.После всех премещений фабричный метод остался пустой, можна сделать его
абстрактным. Если в нем чтото осталость - это будет его реалищацией по умолчанию
====================================================================

==Преимущества
 Избавляет класс от привязки к конкретным классам продуктов.
 Выделяет код производства продуктов в одно место, упрощая поддержку кода.
 Упрощает добавление новых продуктов в программу.
 Реализует принцип открытости/закрытости.
==Недостатки
 Может привести к созданию больших параллельных иерархий классов,
 так как для каждого класса продукта надо создать свой подкласс создателя.
 ===================================================================

 Отношения с другими паттернами:
 *Многие архитектуры начинаються с применения Фабричного метода и эволюционируют
 в Абстрактную фабрику, прототип или Строитель.
 *Классы Абстрактной фабричи часть реализируют с помощью Фабричного мтеода,
 млм на основе Прототипа.
 *Фабричный метод построен на наследовании, но не траебует сложной инициализации.
 *фабричный метод можно рассматривать как часный случай Шаблонного метода, и он
 нередко бывает частью большого класса с Шаблонными методами.
